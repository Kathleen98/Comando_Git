Comando Ação
git init Comando para iniciar o versionamento do projeto, com ele é criada uma pasta oculta ".git" com as configurações necessárias para o controle de versionamento
git status Comando para verificar como está o versionamento do projeto, exemplo, quais alterações, arquivos estão versionados ou não.
git add Comando para adicionar alterações realizadas, pode ser usado o comando "git add ." para adicionar todas as alterações de uma vez ou "git add X" usando o nome do arquivo que deseja adicionar.
git commit -m Comando para realizar comentário antes de subir a alteração.
git commit -a -m "" Comando que une o "git add" + "git commit -M" realizando assim a inclusão de todas as alterações e adicionando comentário.
git branch -m main Com esse comando, renomeamos a branch principal para "main", essa alteração é feita por convenção.
git remote add origin Após esse comando, colocamos a URL do repositório do GitHub para linkar o repositório local, no caso, a pasta do projeto, com o repositório remoto.
git push -u origin main Comando para enviar os arquivos do repositório local para o remoto.
Comandos Ação
git branch Comando para ver as branchs existentes.
git checkout Comando para mudar de branch, é só adicionar o nome da branch após o comando.
git checkout -b Comando para criar uma nova branch
git push --set-upstream origin "nome da branch" Comando para linkar a nova branch ao repositório remoto (após esse comando é possível dar git push)
git merge Comando para unir as branchs, exemplo: Estou na branch main e quero unir com a desenvolvimento "git merge desenvolvimento". Após o merge, é preciso dar "git push" para unificar o código no repositório remoto.

git pull Puxa os arquivo do repositório remoto para o local.
git clone Após esse comando, adicione a URL do repositório que deseja clonar, caso queira trocar o nome da pasto do projeto, dê um espaço e escreva o nome que deseja
git clean -f remove arquivos, ele remove apenas arquivos não adicionados?
git checkout + o nome do arquivo faz com que o arquivo que foi modificado volte para seu ponto de partida, antes de realizar alterações no código.
git log Comando que mostra todo o tracker realizado no repositório, conseguimos visualizar todos os commits
